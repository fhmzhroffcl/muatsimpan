## Your media, in a library you control

**Musim (Muat + Simpan)** turns the things you save into a media library you actually own. Paste one or many links to save video or audio from 1,000+ supported sources, then organize everything with folders, a visual timeline, sticky notes, a built-in player, a two-panel editor, and archive insights—all locally on your computer.

The collection is the home screen. Adding new media is secondary. Your files live in a folder you choose, and Musim shows you exactly where they are.

## Privacy by design

Musim saves files directly to your chosen folder. The current build has no advertising, analytics, telemetry, user accounts, or tracking. It is designed for media you own or are licensed and permitted to keep—your own uploads, public or Creative Commons material, and authorised backups.

Read the [privacy policy](legal/PRIVACY.md) and [responsible-use terms](legal/TERMS.md).

## macOS and Windows

Musim keeps the same local-first archive model on both platforms. The interface and media engine are implemented separately so each version can use the appropriate desktop technologies.

| | macOS | Windows |
| --- | --- | --- |
| Current release | v2.0 | v1.0.0 initial release |
| Application stack | SwiftUI + AppKit | Tauri + Rust + React/TypeScript |
| Player and editor | AVFoundation | Windows media playback + FFmpeg export |
| Package | Apple Silicon DMG | x64 setup installer or portable EXE |
| App support data | `~/Library/Application Support/Musim` | `%APPDATA%\Musim` |
| Signing status | Ad-hoc signed; not notarized | Unsigned pre-release |

The Windows version preserves the archive, library, activity, player, editor, settings, and guide workflows. Small platform-specific details—such as file dialogs, folder reveal actions, notifications, and installation—follow Windows behaviour.

## Download

### macOS

For **Apple Silicon Macs running macOS 14 or newer**:

1. [Download `Musim.dmg`](https://github.com/fhmzhroffcl/muatsimpan/releases/download/v2.0/Musim.dmg).

### Windows

The current Windows release is built for **64-bit Windows** and uses Microsoft Edge WebView2, which is normally included with current Windows 10 and Windows 11 installations.

- [Download the Windows setup installer](https://github.com/fhmzhroffcl/muatsimpan/releases/download/windows-v1.0.0/Musim_1.0.0_x64-setup.exe) for the normal installation experience.
- [Download the portable `musim.exe`](https://github.com/fhmzhroffcl/muatsimpan/releases/download/windows-v1.0.0/musim.exe) to run Musim without installing it.

The first Windows release is unsigned, so Microsoft Defender SmartScreen may ask for confirmation before launch. Review the publisher warning and file checksum on the [Windows release page](https://github.com/fhmzhroffcl/muatsimpan/releases/tag/windows-v1.0.0).

Release checksums are listed on each release page.

## Build from source

For macOS, you need macOS 14 or newer, an Apple Silicon Mac, and Swift Command Line Tools.

```zsh
./scripts/bundle.sh
open build/Musim.app
```

The local packaging script expects third-party executables in `Vendor/`. Those binaries are intentionally excluded from Git. Build and release artifacts, including DMGs, are published separately through GitHub Releases.

For Windows, install Rust and Node.js on a Windows machine, then run:

```powershell
cd windows
./scripts/build.ps1
```

The Windows build fetches yt-dlp, FFmpeg, FFprobe, and Deno during packaging. See the [Windows build guide](windows/README.md) for its architecture, outputs, and GitHub Actions workflow.
